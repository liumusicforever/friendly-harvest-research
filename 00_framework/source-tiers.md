# 來源分級規則（天條）

> 狀態：定稿　　最後更新日：2026-07-14
> 本文件負責問題：什麼可以引用、什麼不能引用、怎樣才算「有佐證」。

## 分級

### Tier 1 — 可直接引用
- 政府漁業／保育主管機關的**現行法規原文**、官方公告、官方管理指引與休閒採捕手冊。
- **同儕審查**期刊論文。
- 國際組織技術文件：FAO、ICES、GFCM、NOAA 等。

### Tier 2 — 可引用，但須在文中標註其性質
- 政府委託研究報告。
- 大學海洋研究中心／研究機構報告。
- 具公信力 NGO 的**技術文件**（非倡議稿、非募款文宣）。

### 排除（一律不得作為主張依據）
- 社群貼文、論壇、部落格、釣具商行銷文、YouTube。
- 無法回溯原始出處的新聞轉述。
  - 例外處理：新聞只能當**線索**。追到它引用的 Tier 1 原始出處後，改引原始出處；
    追不到 → 記為缺口，不引用。

## 佐證三件組（verified 的門檻）

一筆來源要標 `status=verified`，必須同時具備：

| 件 | 欄位 | 說明 |
|---|---|---|
| 1 | `url` | 可公開存取的原始出處（不是搜尋結果頁、不是快取） |
| 2 | `evidence_locator` | 定位到條文層級：`第 41 條` / `p.12` / `Table 2` / `§3.1` |
| 3 | `evidence_quote` | 原文短引 ≤50 字（中／日／英原文照抄），供人工反查 |

三件缺一 → `status=unverified` → **不得**被第 6 階段政策建議引用。

## 法規時效

- 只引**現行版本**。引用時務必檢查修正日期。
- 舊版本若已被取代，registry 該列改標 `status=superseded`，
  **不刪除**（保留研究軌跡），並在新列的 `claim` 註明取代關係。

## 衝突處理

兩份來源說法不一致時：**兩者都留**，在知識層文件中並列標註差異與各自來源編號，
不靜默採信其一。
