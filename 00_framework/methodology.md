# 方法論

> 狀態：定稿　　最後更新日：2026-07-14
> 本文件負責問題：這個研究怎麼推進、每階段的完成定義是什麼。

## 路線：先發散、後整合

```
第 1–2 階段  建地基   台灣現況 + 科學理論（不看任何國家案例）
第 3–4 階段  大發散   各國案例 + 魚槍專題（刻意做寬，寧可多收）
第 5–6 階段  收斂     比較矩陣 + 整合方案
```

**不得跳過發散階段直接寫方案。** 第 6 階段的每一條建議，都必須能指回
第 2 階段的某條科學依據，以及第 3–4 階段的某個國際先例。

## 每階段的固定動作

1. 蒐集 → 對每筆來源執行歸檔四動作（見 `archive-policy.md`）。
2. 撰寫該階段知識層文件，每項主張標 `[S###]`。
3. 更新 `CHANGELOG.md`。
4. 停下來回報：關鍵發現、缺口、下一階段建議。

## 缺口管理

找不到來源時**記為缺口**，不得臆測。缺口以 `【待查證】` 標記，並在該階段文件
末尾的「缺口清單」集中列出，包含：想確認什麼、找過哪裡、下一步可以怎麼找。

## 第 3 階段的固定六問（每個轄區一律回答）

1. 有無最小體長下限？
2. 有無**最大體長上限**？
3. 有無**區間限制（slot limit）**？
4. 休閒 vs 商業是否分流管理？
5. 魚槍有無**專屬規範**？
6. 有無教育／證照制度？

固定六問是為了讓第 5 階段的矩陣能直接拼裝，不必回頭重問。

## 品質檢核（每階段收尾前跑一次）

- [ ] 文件中每個 `[S###]` 都存在於 registry？
- [ ] registry 中本階段新增列，`status` 都是 `verified`？（否則列入缺口）
- [ ] 每筆來源都有 `archive/cards/S###.md` 與本地快照？
- [ ] 素材層有沒有偷渡結論？知識層有沒有無來源的主張？
